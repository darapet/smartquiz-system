How App Updates Work

Follow these steps every time you want to push an update to users

  1. Build a new APK with a higher version In your app-build.gradle file, bump versionCode by 1 and versionName (e.g. 1.1.0 → 1.2.0). Then build the release APK.
  2. Upload the APK to GitHub Releases Go to your GitHub repo → Releases → Draft a new release. Tag it v1.2.0, attach the APK file, then publish. Copy the direct download link to the .apk file.
  3. Fill in the form below and tap "Push Update" Enter the new version, paste the APK link, write what's new — then tap the button. Users with the old version will see the update popup automatically next time they open the app.

🔑 GitHub Token (one-time setup)

Enter your GitHub Personal Access Token with repo write permission. It's saved only on this device — never uploaded anywhere.

✅ Token saved on this device

Create one at: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → tick repo
